Next Steps:
-----------
1. Verify Prometheus is running:
   kubectl get pods -l app=prometheus -n {{ .Release.Namespace }}

2. Check ServiceMonitors are configured:
   kubectl get servicemonitor -n {{ .Release.Namespace }}

3. View PrometheusRules:
   kubectl get prometheusrule -n {{ .Release.Namespace }}

{{- if .Values.alerting.enabled }}
4. Check AlertManager configuration:
   kubectl get alertmanager -n {{ .Release.Namespace }}

{{- if .Values.alerting.webhook.enabled }}
5. Webhook alerts configured:
   Alerts will be sent to: {{ .Values.alerting.webhook.url }}
   Test the webhook by triggering an alert (e.g., HighRequestRate)
{{- end }}

{{- if .Values.alerting.email.enabled }}
5. Email alerts configured:
   Alerts will be sent to: {{ .Values.alerting.email.to }}
   SMTP Host: {{ .Values.alerting.email.smtpHost }}
{{- end }}
{{- end }}

Important Notes:
----------------
- This chart only deploys monitoring components
- The application (frontend and model-service) must be deployed separately using the sms-app chart
- ServiceMonitors expect services in namespace: sms-app
- Update alerting configuration in values.yaml before deploying to production
